## 1.1 公式定義 — BuJo は「ノート術」ではない

Ryder Carroll 公式サイト bulletjournal.com は、BuJo を次のように定義する。

> "The Bullet Journal method is a system that combines elements of mindfulness, productivity, and self-discovery" — designed to help you "track the past, organize the present, and plan for the future."[^bj-faq]

[^bj-faq]: <https://bulletjournal.com/blogs/faq/what-is-the-bullet-journal-method>（アクセス日 2026-05-24）

この定義の重要なポイントは三つある。

1. **マインドフルネス（mindfulness）**: 注意の方向と質に対する自覚
2. **生産性（productivity）**: タスクを実行に移す力
3. **自己発見（self-discovery）**: 自分が「何を」「なぜ」やりたいかを掘り当てる

つまり BuJo は、ToDo リスト・スケジューラ・日記の機能を統合したノート術であると同時に、「自分は何に時間を使うべきか」を能動的に問うためのフレームワークである。「ライフ OS」と呼ばれることがあるのはこのためで、書籍の副題も *Track the Past, Order the Present, Design the Future* となっている[^book]。

[^book]: Ryder Carroll, *The Bullet Journal Method: Track the Past, Order the Present, Design the Future*, Portfolio / Penguin, 2018.

### 公式が強調する「タスク管理アプリではない」という線引き

Carroll は繰り返し、「BuJo は生産性ツールに見えるが、本質はマインドフルネス実践である」と述べている。書籍序章には次のような表現がある（要旨）:

- BuJo の目的は「より多くを成すこと」ではなく、「より少なく、しかし本当にやるべきことを成すこと」である
- 書くこと、それ自体が思考の整流装置である
- アナログ媒体（紙＋ペン）を採用しているのは、その「摩擦」がマインドフルネスを生むから

この点は、デジタル化を考える上で繰り返し戻ってくる前提なので、ここで定着させておく。

## 1.2 考案者 Ryder Carroll — ADHD という起点

### 経歴のあらまし

- 出生: オーストリア・ウィーン[^ted]
- 現在: ニューヨーク・ブルックリン在住のデジタルプロダクトデザイナー
- 役職遍歴: 米大手ファッションブランドで史上最年少のアートディレクター、起業家、NYC のデザイン会社の責任者[^bj-rc]

[^ted]: TEDxYale 2017, "How to Declutter Your Mind — Keep a Journal." <https://www.youtube.com/watch?v=ym6OYelD5fA>
[^bj-rc]: <https://bulletjournal.com/pages/rydercarroll>

Carroll は「デジタル製品をつくる人」がアナログのノート術を世に問うた、という構図がメディアでよく取り上げられる。これは皮肉ではなく、彼が**両方の世界を知っているからこそ「あえてアナログ」を選んだ**ことの裏返しでもある。

### ADHD と BuJo の関係

これは BuJo を理解する上で**最重要**の出自情報である。Carroll は子供のころから ADHD（注意欠陥多動性障害）であり、当時は今のように支援リソースが整っていなかった。彼は自伝的 Medium 記事 *Inside ADHD* で次のように書いている。

> "Having ADHD is like trying to catch the rain."[^adhd]

[^adhd]: Ryder Carroll, "Inside ADHD," *Human Parts (Medium)*, 2019-10-16. <https://humanparts.medium.com/inside-adhd-55b9618cd708>

雨を素手で受け止めようとしているような注意力の散漫さ。教師や同級生からは「頭の悪い子」と扱われ、procrastination（先延ばし）、hyperactivity（過活動）、overwhelm（情報過多による麻痺）、不安が日常だった[^bj-rc]。

20 年以上にわたり、彼はノートとペンだけで自分を機能させるためのシステムを試行錯誤した。

- 初期は単純な ToDo リスト
- うまくいかず、複数のリストを同居させる工夫を加え
- それでも書き写しコストが膨らむため、**最小限の記号で全種類のエントリを表現する**方向に進化
- やがて「タスクは `・`、イベントは `○`、ノートは `–`」という極小の記法に収束した

つまり、いま我々が見る BuJo の記号体系（`・` `○` `–` `×` `>` `<` `*` `!`）は、**「ADHD の脳でも続けられる最低限のルール」として淘汰圧でできた**ものである。これは、デジタル化する際に「シンプルさをどこまで損ねないか」を考えるときの重い前提になる。

### 「The Why」への気づき

20 代後半〜30 代の Carroll は、ノート術によって職業的には成功した。しかし彼自身の表現によれば「存在的に空っぽ（existentially empty）」になっていた[^bj-rc]。

> "I had been pursuing other people's goals, not my own."

ここで彼は、「タスクをこなす」という機能から一歩戻り、「**なぜ**それをやるのか（The Why）」を問う構造を BuJo に組み込んだ。これが後述の「マイグレーション」「リフレクション」のコア哲学になり、書籍 *The Bullet Journal Method* の中心テーマになった。

## 1.3 公開と普及 — 2013 → 2018 → 現在

### タイムライン

| 年 | 出来事 |
|---|---|
| 1990s 末 | Carroll 大学時代に原型を完成（諸説あるが、最も早い言及は大学時代）[^wiki] |
| 2013 | YouTube に "How to Bullet Journal" 動画を公開、バイラル化[^wiki] |
| 2015〜 | Instagram / Pinterest を中心に「美しい BuJo」が爆発的に拡散 |
| 2017 | TEDxYale で "How to Declutter Your Mind" を講演[^ted] |
| 2018 | Penguin / Portfolio から *The Bullet Journal Method* を出版、NYT ベストセラー入り[^book] |
| 2018 | Instagram の `#bulletjournal` 投稿数が 300 万件超に[^wiki] |
| 〜現在 | 公式アプリ "Bullet Journal Companion" の提供、Edition 2 公式ノートの発売、28 言語に翻訳[^bj-book-page] |

[^wiki]: "Bullet journal," *Wikipedia*, accessed 2026-05-24. <https://en.wikipedia.org/wiki/Bullet_journal>
[^bj-book-page]: <https://bulletjournal.com/pages/book>

### Kickstarter

Carroll は BuJo の専用ノート（後述する Leuchtturm1917 とのコラボ版「Bullet Journal Edition 2」とは別系統の初期試み）で Kickstarter を起こし、**約 8 万ドル**を集めたとされる[^wiki]。これは「ノート 1 冊を売る」という極めて素朴なプロダクトに対する集金額として、当時かなり象徴的な数字だった。

### バイラル化の理由 — Pinterest / Instagram / TikTok

BuJo を世界的トレンドに押し上げたのは、Carroll の方法論そのものよりも、**コミュニティが二次創作した「装飾系 BuJo（artistic / decorative BuJo）」**である。Pinterest や Instagram で「美しい見開きページ」「水彩画のような habit tracker」「凝った字体のヘッダー」が拡散し、これが BuJo の表象となった。

ここで強調しておくべきは:

- **Carroll 自身は「装飾」を強制していない**
- 公式のオリジナル動画は極めて質素なノートで、見出しに飾りもなく、字も整っていない
- 彼は後年、「Instagram の影響で BuJo の本来の意味が見失われている」と苦言を呈している（→ `09-community-and-variants.md` で詳述）

この「**Carroll が言っていない BuJo**」と「**Carroll が言っている BuJo**」の差は、デジタル化アプリの方針設定で**最初に決めるべき選択肢**になる。

## 1.4 書籍 *The Bullet Journal Method*（2018）の位置づけ

### 書誌情報

- 著者: Ryder Carroll
- 出版社: Portfolio / Penguin Random House
- 出版年: 2018 年 10 月
- ISBN: 978-0-525-53333-7
- 副題: *Track the Past, Order the Present, Design the Future*
- 翻訳: 28 言語以上（日本語訳『バレットジャーナル 人生を変えるノート術』ダイヤモンド社, 2019）[^bj-book-page]

### 内容構成（章立ての要旨）

書籍は大きく 4 部構成。

1. **The Preparation**（準備）: なぜ書くのか、なぜアナログなのか、注意力をめぐる現代的問題
2. **The System**（システム）: Rapid Logging、Index、Future/Monthly/Daily Log、Collections、Migration の運用
3. **The Practice**（実践）: 始め方、目標設定、習慣の作り方、不安や倦怠への向き合い方
4. **The Art**（応用）: コミュニティ事例、よくある悩みへの対応、ライフプロジェクトへの拡張

ここで重要なのは、書籍が**「使い方」だけでなく「思想」に大きく割いている**ことだ。実際、「The Practice」「The Art」のページ数が「The System」より多い。

> "Part productivity system, part mindfulness practice."[^bj-book-page]

これは公式が一貫して伝えているメッセージで、書籍はその思想面の弁明書として機能している。

### 代表的なエンドースメント

- Cal Newport（*Deep Work* 著者）: "One of the most elegant and effective productivity systems"[^bj-book-page]
- Tiago Forte（*Building a Second Brain* 著者）: "Taking action, clarifying your values, and focusing your attention on what matters most"[^bj-book-page]

両者とも、BuJo を**「単なる ToDo」ではなく「価値観の明確化と注意の方向づけ」**として位置づけているのが特徴。

## 1.5 BuJo が解こうとしている問題

### Carroll の問題定義（TEDxYale 2017 要旨）

> "Being busy doesn't mean you're being productive. A lot of times, being busy just means you're in a state of being functionally overwhelmed."[^ted-ideas]

[^ted-ideas]: "How to Declutter Your Mind," *ideas.ted.com*. <https://ideas.ted.com/how-to-declutter-your-mind/>

彼の問題定義は次の三層構造になっている。

1. **情報過多（overwhelm）**: 現代人は注意を奪うものに浸かっており、頭の中だけでは整理しきれない
2. **目的の喪失（Why の不在）**: タスクを処理することと、有意義に生きることは別であり、ToDo を増やしても後者は満たされない
3. **持続性の欠落（unsustainable systems）**: 多くの生産性ツールは初期コストが高すぎ、結局続かない

BuJo はこの三つを、それぞれ次の機構で解こうとしている。

| 問題 | BuJo の機構 |
|---|---|
| Overwhelm | **Rapid Logging**（短文で素早く外部化） + **マイグレーション**（雑音と本質を分離） |
| Why の不在 | **AM/PM リフレクション**、**マイグレーション時の問い直し**、書籍にある「Why の 5 階層」 |
| Unsustainable | **記号 3 種だけの極小ルール**、**装飾不要**、**ノート 1 冊で完結** |

### ここからアプリに落ちる含意（プレビュー）

詳細は `10-digital-implementation-considerations.md` で扱うが、最初の見取り図として:

- 「タスクを早く入力できる UI」と「タスクと向き合う UI」は別物として設計する必要がある
- 「マイグレーション」は機能ではなく**儀式**として扱う必要がある（=スワイプで一括移動 ≠ BuJo）
- 「装飾」は本筋ではない（ただし、ユーザー獲得のためには無視できない側面でもある）

## 1.6 なぜ普及したのか — 仮説とエビデンス

Wikipedia と各種メディアが言及している普及要因を整理する[^wiki]。

### 1) シンプルさが極端

`・` `○` `–` の三記号と「ノート 1 冊」だけで成立する。これは新しい習慣を学ぶ際のコストとして極めて低い。

### 2) アナログ回帰のタイミング

2010 年代後半は、スマホアプリ疲れ・ソーシャル疲れに対して「ペンとノート」が反動的に注目された時期と重なる。BuJo はそのアイコンになった。

### 3) インスタグラム適合性

皮肉なことに、Carroll が後で批判することになる「装飾系 BuJo」が Instagram のビジュアル文化に強烈にハマった。`#bulletjournal` は 300 万件超の投稿を生み[^wiki]、TikTok でも継続的に新しい層を取り込んでいる。

### 4) ADHD コミュニティでの強い支持

公式は ADHD と BuJo の親和性を積極的に語っており、ADHD 当事者団体や ADHD コーチが繰り返し推奨してきた[^totallyadd]。これは「自分の脳のために設計された」という強い説得力を持つ。

[^totallyadd]: "Can A Bullet Journal Help Me Get Organized?" *TotallyADD*. <https://totallyadd.com/bullet-journal/>

### 5) 書籍によるレジビティの獲得

2018 年の書籍出版で、それまで「YouTube とブログで流行ったやつ」だった BuJo が「ベストセラー著者の正統メソッド」に格上げされた。これにより、職場や教育機関での導入が増えた。

---

### 6) 「自分で作る」余地

BuJo は完成品ではなく「空っぽの家」（Carroll の比喩、→ `09`）として提供された。ユーザーは自分の Custom Collection を作り、自分の Signifier を足し、自分のレイアウトを工夫できる。この「**自分のシステムを所有する**」感覚が、強い帰属意識とコミュニティの自己増殖を生んだ。市販プランナーの「決められた枠に従う」体験とは対照的。

## 1.6.5 BuJo の全体像を 1 枚で — 時間スケール別の動き

BuJo を初めて理解するとき、最も混乱するのが「いつ、何をするのか」である。時間スケールで整理すると、全体像が掴める。

```
┌─────────────────────────────────────────────────────────────────┐
│ スケール    やること                          関連コレクション  │
│ ─────────  ──────────────────────────────────  ─────────────────  │
│ 随時       Rapid Logging（思いついたら書く）   Daily Log         │
│             ・ Task / ○ Event / – Note を即記録                  │
│                                                                 │
│ 毎朝       AM Reflection（今日の準備）          Daily/Monthly Log│
│             今日のページを作り、優先を * で示す                  │
│                                                                 │
│ 毎晩       PM Reflection（1 日のクロージング）  Daily Log        │
│             × を付け、書き忘れを追加、気づきを –                 │
│                                                                 │
│ 月末/月初  Monthly Migration（選別の儀式）      Monthly/Future   │
│             未完了タスクを 1 件ずつ判定し書き写す                │
│                                                                 │
│ ノート満杯 Yearly Migration（総決算）           新ノート全体     │
│             1 年を振り返り、何を続け何を捨てるか決める           │
└─────────────────────────────────────────────────────────────────┘
```

この「随時 → 毎朝 → 毎晩 → 月次 → 年次」という**入れ子のサイクル**こそが BuJo の運用骨格である。デジタル化では、このサイクルのどこに「速さ」を、どこに「摩擦」を配置するかが設計の肝になる（→ `06`, `10`）。

### 「タスクアプリ」と決定的に違う点（早見表）

| 観点 | 一般的なタスクアプリ | BuJo |
|---|---|---|
| 扱う対象 | Task のみ | Task / Event / Note の混在 |
| 未完了タスク | 自動で繰り越し | 手で 1 件ずつ書き写し（=選別） |
| 振り返り | 任意・後付け | 朝晩・月次・年次に組み込み |
| 「なぜやるか」 | 通常は扱わない | Why を問う構造が中心 |
| 評価軸 | 完了率・効率 | 意図性・自己理解 |
| 装飾 | ない | 任意（コミュニティでは盛ん） |
| データ | クラウド前提 | アナログ前提（紙 1 冊） |

この表の右列を「便利にしすぎて左列に寄せてしまう」のが、デジタル化の最大の失敗パターンである。

## 1.7 まとめ — 本ドキュメント全体の出発点

ここまでの要点を、後続ドキュメントの導入として整理しておく。

1. BuJo は **「ノート術 + マインドフルネス + 自己発見」の統合システム**で、単なる ToDo ではない
2. 起源は **ADHD の Carroll が自分のために作った最小ルール**であり、シンプルさは設計思想
3. 2013 年公開、2018 年書籍化で世界的普及。**Carroll が言うメソッド**と **コミュニティが派生させた装飾文化**は別物
4. デジタル化を考える時、最初の問いは「**どちらの BuJo を作るのか**」である

次は `02-philosophy.md` で、この「マインドフルネス + 自己発見」の哲学面を深掘りする。これを理解しないと、なぜマイグレーションを自動化してはいけないか、なぜリフレクションを省略してはいけないか、が分からない。

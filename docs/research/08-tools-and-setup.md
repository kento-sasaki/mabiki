# 08. 必要な道具とセットアップ — ノート、ペン、ドット方眼

## このドキュメントで分かること

- BuJo に必要な最小限の道具（公式の立場）
- ノートの選び方（Leuchtturm1917 等が定番である理由）
- ドット方眼（dot grid）が選ばれる理由
- 新しいノートの初期セットアップ手順

**想定読者**: 物理的な BuJo のリアリティを掴みたい実装者。デジタルアプリで「物理ノートの体験のどこを再現するか」を判断する人。

---

## 8.1 公式の立場 — 「ノートとペンがあればいい」

### 公式の指示

> "Begin with a notebook and something to write with."[^how-to]

[^how-to]: <https://bulletjournal.com/pages/how-to-bullet-journal>

Carroll の立場は徹底して**ミニマル**。彼は「どのノートを選ぶか」よりも「**どう使うか**」が重要だと繰り返す。

> "The method prioritizes how you use your notebook rather than which one you select."[^how-to]

つまり、高級ノートも文房具も**必須ではない**。これは ADHD 起源の「ハードルを下げる」設計思想と一致する。コストが高い道具を要求すると、続かない人が増える。

### しかし、定番は存在する

公式が「何でもいい」と言いつつ、コミュニティと公式コラボで定番化した道具がある。これを知っておくと、デジタル化で「ユーザーがどんな物理体験を持っているか」が分かる。

## 8.2 ノート — Leuchtturm1917 が定番である理由

### 定番ノート一覧

| ブランド | 特徴 | 価格帯 |
|---|---|---|
| **Leuchtturm1917** | 業界標準。ページ番号・目次・dot grid が最初から印刷済み | 中〜高 |
| Rhodia | 紙質が良い、万年筆向き | 中 |
| Clairefontaine | 滑らかな紙、裏抜けしにくい | 中 |
| Moleskine | 知名度高いが紙質は賛否 | 中 |
| Field Notes | ポケットサイズ、軽量 | 低 |
| Zequenz | しなやかに開く | 中 |

[^tinyray]: <https://www.tinyrayofsunshine.com/blog/bullet-journal-guide>

### なぜ Leuchtturm1917 が定番なのか

[^leuchtturm]: <https://www.leuchtturm1917.us/bullet-journal-edition-2.html>

1. **ページ番号が最初から印刷されている**: BuJo は全ページに番号が必要だが、自分で書くのは面倒。Leuchtturm1917 は最初から振ってある
2. **目次ページが最初から付いている**: Index に使える空欄ページが冒頭にある
3. **dot grid（ドット方眼）が標準**: BuJo に最適な罫線
4. **アーカイブ品質の紙**: 万年筆でも裏抜けしにくい
5. **A5 サイズが標準**: 持ち運びと書き込みのバランス

### 公式コラボ「Bullet Journal Edition 2」

Carroll と Leuchtturm1917 が共同開発した公式ノート。特徴[^bj-edition2]:

[^bj-edition2]: <https://bulletjournal.com/blogs/bulletjournalist/edition-2>

- **ページ番号が中央配置**: Threading で見開きを参照するとき読みやすい
- **Key（凡例）ページが印刷済み**: 記号の意味が最初から書いてある
- **Index・Future Log のテンプレートが印刷済み**
- **使い方ガイドが綴じ込み**
- **3 本のしおり紐**: Index / Monthly / Daily をすぐ開ける

このノートは「**BuJo メソッドを物理製品に固めたもの**」であり、デジタルアプリの仕様を考えるとき、リファレンス実装として参考になる。

## 8.3 ドット方眼（dot grid）が選ばれる理由

### 4 つの罫線スタイル

Leuchtturm1917 等のノートには 4 種類のページがある[^jetpens]。

[^jetpens]: <https://www.jetpens.com/blog/Leuchtturm1917-Notebooks-A-Comprehensive-Guide/pt/934>

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│   Dotted     │   Squared    │    Ruled     │    Plain     │
│  (ドット方眼)│  (方眼)      │  (横罫)      │  (無地)      │
│              │              │              │              │
│  · · · · ·   │  ┼─┼─┼─┼     │  ─────────   │              │
│  · · · · ·   │  ┼─┼─┼─┼     │  ─────────   │   (空白)     │
│  · · · · ·   │  ┼─┼─┼─┼     │  ─────────   │              │
└──────────────┴──────────────┴──────────────┴──────────────┘
```

### なぜドット方眼か

ドット方眼が BuJo の定番になった理由:

1. **構造のガイドになるが、邪魔しない**: 罫線（ruled）は文章には良いが、表やトラッカーを描くと窮屈。方眼（squared）はグリッドが目立ちすぎ。ドットは「**必要なときだけ目印になる**」中庸
2. **直線も曲線も描ける**: ドットを基準に縦横の線を引ける、かつ無地に近い自由度がある
3. **カレンダー・トラッカーを描きやすい**: Monthly Log の Calendar Page や Habit Tracker のグリッドを作りやすい
4. **手書き文字を邪魔しない**: 薄いグレーのドットは、書いた文字より目立たない

### デジタル化での示唆

ドット方眼の「**構造化しつつ自由**」という性質は、デジタル UI の設計思想として参考になる。

- 完全に自由なテキストエリア（無地）= 構造がなく、続かない
- 完全に固定されたフォーム（方眼）= 窮屈で、BuJo の柔軟性が消える
- **中庸**: ゆるい構造（Bullet 種別だけ決まっている、あとは自由）

これは「Bullet 入力欄は構造化（種別選択あり）するが、内容は自由テキスト」というデザインに対応する。

## 8.4 ペン — 何でもいい、が定番はある

### 公式の立場

ペンも「何でもいい」。ただし、コミュニティで定番化したものはある。

| ペン | 特徴 |
|---|---|
| Sakura Pigma Micron | 耐水性、裏抜けしにくい、細字バリエーション豊富 |
| Pilot Metropolitan | 万年筆、書き味が良い |
| Faber-Castell | 製図用、にじみにくい |
| Staedtler Triplus Fineliner | カラフル、装飾系に人気 |
| Pilot G-2 | 安価で入手しやすいゲルインク |

### 装飾系 vs ミニマル系の分岐点

ペン選びは、`09-community-and-variants.md` で扱う「装飾系 vs ミニマル系」の分岐とも関わる。

- **ミニマル系**: 黒 1 本で十分
- **装飾系**: 複数色、ブラッシュペン、蛍光ペン、マスキングテープ等

Carroll 本人は「黒ペン 1 本」スタイルに近い。公式動画でも装飾はほぼない。

## 8.5 その他の道具（任意）

コミュニティで使われる補助道具:

| 道具 | 用途 |
|---|---|
| 定規 | カレンダーやトラッカーの線を引く |
| ステンシル | 月名、アイコン等を綺麗に |
| マスキングテープ | ページの装飾、区切り |
| ステッカー | 装飾系で人気 |
| しおり紐 | Index / Monthly / Daily にすぐアクセス |
| ペンホルダー | ノートにペンを留める |

これらは**全て任意**。Carroll の立場では「効果があるなら使え、なければ捨てろ」。

## 8.6 初期セットアップ手順（公式準拠）

新しいノートを開いてから最初に行うこと。

```
┌─────────────────────────────────────────────────────────┐
│ STEP 0: ページ番号                                      │
│   Leuchtturm1917 なら印刷済み。なければ自分で振る       │
│   (最初に全部振らず、使うたびに振ってもよい)            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ STEP 1: Key（凡例）ページ — 任意                        │
│   記号の意味を 1 ページに書いておく                     │
│   ・ Task / ○ Event / – Note / × Done / > Migrated 等   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ STEP 2: Index — 必須                                    │
│   最初の 2-4 ページを Index として確保                  │
│   (まだ空欄。後から書き足す)                            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ STEP 3: Future Log — 推奨                               │
│   次の 2-4 ページに 6-12 ヶ月分の枠を作る               │
│   (横線を引いて月を区切る)                              │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ STEP 4: 最初の Monthly Log                              │
│   見開き 2 ページ                                       │
│   左 = Calendar Page (日付と曜日を縦に)                 │
│   右 = Task Page (今月のタスク)                         │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ STEP 5: 最初の Daily Log                                │
│   次のページから、日付を Topic にして Rapid Logging 開始│
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ STEP 6: Index を更新                                    │
│   作った各コレクションを Index に登録                   │
│   Future Log ... 3-4 / May 2026 ... 5-6 等              │
└─────────────────────────────────────────────────────────┘
```

### 重要な原則

- **全部を最初に作らない**: Index と最初の Monthly Log だけ作れば、あとは使いながら追加
- **完璧を求めない**: 最初のセットアップに時間をかけすぎると続かない
- **数分で始められる**: 「ノートとペンを持って 10 分」で運用開始できることが設計目標

## 8.7 「最初のページ」をめぐる思想

### Carroll の助言

書籍では、新しいノートを前にして「完璧に始めたい」という誘惑が、続かない最大の原因だと指摘する。

- 美しい表紙ページを作ろうとして、何週間も始められない
- Index を完璧にしようとして、迷う
- 罫線を完璧に引こうとして、消しゴムで消す

→ **「とりあえず始めろ」**が公式のメッセージ。間違いも、汚い字も、BuJo の一部。

### デジタル化での示唆

「最初のページの心理的ハードル」は、アプリのオンボーディングでも再現される問題。

- 設定が多すぎると始められない
- テンプレートが豪華すぎると気後れする
- **「最初の 1 エントリを 10 秒で書ける」**ことが、アナログの「ノートとペンで 10 分」に対応する

## 8.8 物理ノートの「資産性」

### ノートは消えない

物理ノートには、デジタルにない特性がある。

- **物理的に残る**: 5 年後も本棚にある
- **過去が触れる**: パラパラめくると過去の自分に会える
- **改ざんできない**: 書いたものは（打ち消し線を引いても）残る
- **所有感がある**: 「自分の人生の記録」という重み

### デジタル化で失われやすいもの

- データはクラウドに消えうる（サービス終了、アカウント凍結）
- 編集・削除が容易すぎて「歴史」が残らない
- 「めくる」体験がない

これらは `10-digital-implementation-considerations.md` の主要論点。デジタルでは「**所有感**」「**永続性**」「**改ざん不能性**」をどう設計するかが課題になる（エクスポート機能、編集履歴、削除を打ち消し線で表現する等）。

---

## 8.9 コストの観点

### アナログのコスト

- ノート: 2,000〜4,000 円（半年〜1 年持つ）
- ペン: 数百円〜
- 合計: 年間数千円

### この安さが普及の一因

`01-overview.md` で触れた通り、「ノート 1 冊」という低コストが普及を後押しした。デジタルアプリの価格設定（サブスク等）を考えるとき、「ユーザーは年間数千円のアナログ体験を、何に置き換えるのか」という比較が背景にある。

## 8.10 まとめ

1. 公式の立場は **「ノートとペンがあればいい」**（ミニマル）
2. ただし **Leuchtturm1917**（ページ番号・目次・dot grid 標準）が定番
3. **ドット方眼**は「構造化しつつ自由」で、デジタル UI 設計の参考になる
4. 初期セットアップは **Index + 最初の Monthly Log** だけで始められる
5. 物理ノートの **所有感・永続性・改ざん不能性** は、デジタル化で意識的に再現すべき要素
